# 在服务器创建密钥对
1.首先在服务器中创建有古砚github的密钥对，是和github的actions相关联

具体执行命令：
ssh-keygen -t rsa -b 4096 -C "github-actions-deploy-key"
ssh-keygen：这是在 Linux、macOS 或其他类 Unix 系统中生成新 SSH 密钥对的工具
-t rsa：定义要生成的密钥类型。rsa 是一种广泛使用的加密算法
-b 4096：指定密钥的位数 (bits)。4096 是目前推荐的安全长度
-C "..."： 提供一个注释 (comment) 来帮助您识别这对密钥的用途

2.一直按回车确认即可

3.cat ~/.ssh/github-actions-deploy-key.pub >> ~/.ssh/authorized_keys
* 这行命令是将我们刚才生成的公钥追加到.ssh/authorized_keys文件；
 在执行该命令的时候会提示公钥文件找不到，因为我们在按回车的时候是不会设置存放路径，使用了默认名称 id_rsa（私钥） 和 id_rsa.pub（公钥），而不是使用 github-actions-deploy-key 这个自定义名称；
 如果像使用自定义名称在控制带提示设置存放路径的时候，这样设置
  Enter file in which to save the key (/root/.ssh/id_rsa): ~/.ssh/github-actions-deploy-key 
* 尽然文件名称使用了默认的名称，我们就执行下方命令：
  cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys，把系统生成的默认文件重新追加搭到authorized_keys

  chmod 600 ~/.ssh/authorized_keys
* 这条命令成功地将 authorized_keys 文件的权限更改为 600。这意味着只有文件所有者（root 用户）可以读写此文件。这是 SSH 要求的正确安全权限设置

* 这两步可以一起执行
  cat ~/.ssh/github-actions-deploy-key.pub >> ~/.ssh/authorized_keys
  chmod 600 ~/.ssh/authorized_keys

注释：authorized_keys文件不是服务器自带，如果您运行 cat ... >> authorized_keys 时文件尚不存在，Linux 的 >>（追加重定向）操作符会自动为您创建这个文件，并将内容写入其中。

4.cat ~/.ssh/id_rsa
这一步是查看私钥文件内容，复制私钥，用于和github创建Actions创建联系的时候使用
