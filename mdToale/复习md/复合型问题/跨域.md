跨域是浏览器基于同源策略的安全限制，当请求的协议、域名或端口任一不同时，浏览器会拦截响应数据以保护用户信息安全
同源策略要求协议(http/https)、域名(如example.com)和端口(如8080)必须完全相同，否则视为跨域请求
解决：
    1.利用<script>标签不受同源策略限制的特性，通过动态创建脚本获取跨域数据。仅支持GET请求，需后端配合返回回调函数包裹的数据
         const script = document.createElement('script')
         script.src = 'http://api.com/data?callback=handleResponse'
         document.body.appendChild(script)
    2.修改package.json‌：简单配置"proxy": "http://localhost:5000"，但只能代理单一目标
    3.生产环境Nginx反向代理
      在nginx.conf中添加server块，关键配置包括！
      
server {
    listen 80;
    server_name your_domain.com;
    
    # 静态资源服务
    location / {
        root /path/to/react/build;
        try_files $uri /index.html;
    }
    
    # API反向代理
    location /api/ {
        proxy_pass http://backend_server:port/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
